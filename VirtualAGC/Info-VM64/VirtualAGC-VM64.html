What's this Virtual Machine?

This Virtual Machine (VM), called VirtualAGC-VM64, is intended to provide an easy and portable way to work with the materials from the Virtual AGC Project, and to update it, without the inconvenience of having to deal with the quirks of the various computer platforms (Windows, Mac OS, Linux, et al., in multiple operating-system versions).  I won't describe the Virtual AGC Project here, since if you don't have a pretty good idea already what it is, then you can be assured that you don't need this VM at all and can stop reading right here.

The VM itself is running a light-weight version of the 64-bit Ubuntu Linux operating system that's known as Lubuntu.  The VM can be run on any computer for which there's a version of the VirtualBox virtualization environment, but you likely know that already since you're viewing this message.

Unfortunately, in this VM you cannot actually fly simulated Apollo missions using the Orbiter/NASSP. That's because Orbiter is compatible with Microsoft Windows only, and cannot be run on a Linux system. Nor can I freely provide a Windows-based VM on which to run Orbiter, because unlike Linux, Microsoft Windows cannot legally be freely distributed.

Working with Lubuntu

You're unlikely to be familiar with Lubuntu, but be confident that unfamiliarity won't matter very much to you.  Since this VM is not intended to be a full-fledged system for you to do your daily work or have your entertainment, it doesn't have much software installed on it that isn't directly relevant to the Virtual AGC Project.  But it doesn't need much, and hopefully this short section will familiarize you almost everything you do need to know.

  • As delivered, the desktop is configured so that a single click on any file opens it, similar to a web browser. If you prefer double clicking, as is the default for most operating systems, you can easily configure it. Open the file-system navigator by clicking on the icon on the toolbar at the bottom of the desktop. Use Edit/Preferences on the navigator's menu, and uncheck the box labeled "Open files with single click".
  • Lubuntu's main menu is opened by using the icon that looks like  in the lower-left corner of the screen.
  • Command-line window: There are several different command-line terminals you can use (QTerminal, UXterm, Xterm).  They are all found on the Lubuntu main menu under the heading "System Tools".  I would recommend using QTerminal.
  • Web browser: Firefox is found on the main menu under the heading "Internet".
  • Text editor: Several are installed (FeatherPad, Vim), on the Lubuntu main menu under the heading "Accessories", or obtained by right-clicking on a text file.  I would recommend FeatherPad for ease of use, but some people may prefer Vim its more-advanced capabilities.
  • Lubuntu updater: To update Lubuntu itself, you can use the Lubuntu Update under the "Preferences" heading of the Lubuntu main menu. However, any updates to the C/C++ compilers, to wxWidgets, or to any other package required by Virtual AGC's build system could have the unintended consequence of future failures when trying to update Virtual AGC. Therefore, it is better not to update Lubuntu until/unless you see a notification on the Virtual AGC website that it's desirable to do so.
  • Screenshots and recordings: I'd recommend using the VirtualBox View menu to do that rather than using Lubuntu's facilities, which may not have been installed.
  • System snapshots: If you have configured the VM how you like it and are considering doing anything which might adversely affect it (such as performing a Lubuntu update), you might want to make a snapshot of your current VM configuration before doing so. I would recommend using VirtualBox's snapshotting capability than attempting to install or use any native Lubuntu facilities for that purpose.
